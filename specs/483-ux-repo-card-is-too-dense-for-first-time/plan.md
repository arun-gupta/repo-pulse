# Implementation Plan: Repo Card Progressive Disclosure

**Branch**: `483-ux-repo-card-is-too-dense-for-first-time` | **Date**: 2026-04-27 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/483-ux-repo-card-is-too-dense-for-first-time/spec.md`

## Summary

Add progressive disclosure to the repo card: a default "primary tier" (health score + 3 ecosystem tiles) and an expandable "secondary tier" (dimension tiles, lenses, raw stats, recommendations). Per-repo expand state persists in localStorage. When a CNCF readiness score is available, surface it as a compact badge in the card header rather than adding a bottom section.

## Technical Context

**Language/Version**: TypeScript 5.x / Next.js 14+ (App Router)  
**Primary Dependencies**: React 19, Tailwind CSS, Vitest, Playwright  
**Storage**: Browser localStorage (client-side only, no server state)

**Scale/Scope**: 1 component (`MetricCard.tsx`), 2 test files updated

## Implementation Approach

### Step 1 — TDD: Write failing Vitest tests for progressive disclosure

Add to `MetricCard.test.tsx`:

1. **Default state**: health score and ecosystem tiles visible; score cells, lenses, details, recommendations hidden; "Show details" button present with `aria-expanded="false"`
2. **After expand click**: all secondary content becomes visible; button reads "Hide details" with `aria-expanded="true"`
3. **After collapse click**: secondary content hidden again
4. **localStorage persistence — expand**: expand, re-render with same repo key → starts expanded
5. **localStorage persistence — collapse**: collapse, re-render → starts collapsed
6. **localStorage failure**: `localStorage.getItem` throws → card starts collapsed, no error
7. **No secondary content**: card with no score cells / lenses / details / recommendations → affordance hidden (no point showing "Show details" if there's nothing to show)

### Step 2 — Implement progressive disclosure in MetricCard.tsx

Changes to `MetricCard.tsx`:

1. Add `detailsExpanded` state initialized from localStorage (try/catch)
2. Add `toggleDetails` handler: flips state + writes localStorage (try/catch)
3. Move `scoreCells` render block, lenses block, details `<section>`, and recommendations block inside `{detailsExpanded && …}`
4. Insert "Show details / Hide details" button between ecosystem tiles and the secondary tier gate
   - `data-testid="details-toggle-{card.repo}"`
   - `aria-expanded={detailsExpanded}`
   - `aria-controls="secondary-{card.repo}"`
   - `min-h-[44px]` (touch target compliance — FR-010)
5. Wrap secondary content in `<div id="secondary-{card.repo}">` for `aria-controls`
6. Add "percentile rank" label section header only in secondary tier (currently it sits above the merged ecosystem + score cell grid; move it to secondary tier since ecosystem tiles will have their own visual context)

### Step 3 — TDD: Write failing Vitest tests for CNCF header badge

Add to `MetricCard.test.tsx`:

1. **With aspirantResult**: render card with `card.analysisResult.aspirantResult = { readinessScore: 75, … }` → `data-testid="cncf-badge-{repo}"` is in the document within the header button
2. **Without aspirantResult**: render card with `aspirantResult = null` → `cncf-badge-*` not in document
3. **Badge visible when pane is not collapsed**: CNCF badge is inside the header button (always visible)

### Step 4 — Implement CNCF header badge in MetricCard.tsx

Changes to `MetricCard.tsx`:

1. Import `CNCFReadinessPill` from `@/components/overview/CNCFReadinessPill`
2. Read `aspirantResult = card.analysisResult.aspirantResult ?? null`
3. Inside the header `<button>`, after `<h3>`, before `<p>Created:…</p>`, add:
   ```tsx
   {aspirantResult ? (
     <span data-testid={`cncf-badge-${card.repo}`} aria-label={`CNCF Sandbox Readiness: ${aspirantResult.readinessScore} / 100`}>
       <CNCFReadinessPill aspirantResult={aspirantResult} onClick={…navigate to CNCF tab…} />
     </span>
   ) : null}
   ```
   The `onClick` navigates to the `cncf-readiness` tab using the same `document.querySelector` pattern used by score cells.

### Step 5 — Playwright E2E smoke test

Create `tests/e2e/metric-card-progressive-disclosure.spec.ts` (or add to existing card E2E test):

1. Load page with mocked analysis for `facebook/react`
2. Assert health score visible
3. Assert "Activity" tile NOT visible (secondary tier hidden)
4. Click "Show details" button
5. Assert "Activity" tile visible
6. Reload page
7. Assert "Activity" tile visible (localStorage restored expanded state)

### Step 6 — Regression pass

Run: `npm test`, `npm run lint`, `npm run typecheck`. All existing tests must pass.

## Key Risk: Existing tests expect secondary content visible by default

The existing `MetricCard.test.tsx` has 354 lines of tests that assert things like `screen.getByText(/^Activity$/)` without first clicking "Show details". After implementing progressive disclosure, those assertions will fail because Activity is now in the secondary tier.

**Resolution**: Each affected test must be updated to call `userEvent.click(screen.getByTestId('details-toggle-facebook/react'))` before asserting secondary-tier content. This is expected churn — the tests are being updated to match the new intended behavior, not worked around.

**Scope**: Approximately 8–12 existing test blocks will need the expand-click prefix added. This is tracked in tasks.md.
