# Requirements Checklist: Repo Comparison

- [x] `Comparison` is defined as a real feature, not a placeholder
- [x] Comparison is limited to successful repository analyses only
- [x] Comparison is driven entirely from `AnalysisResult[]`
- [x] Anchor repo is user-selectable
- [x] First successful repo is the default anchor
- [x] Sections are clearly defined and user-toggleable
- [x] Attributes are user-toggleable and default to all selected
- [x] Median column is user-toggleable and defaults to on
- [x] Every visible column is sortable ascending/descending
- [x] Explicit unavailable states are preserved as `—`
- [x] Comparison cap of 4 repos is specified and must be communicated clearly
- [x] Export reuse is acknowledged without making export part of this feature
